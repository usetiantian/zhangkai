## 一、产品定义

### 一句话

**每个人的专属 AI，越用越懂他，能满足他的任何需求。**

### 拆解

| 关键词 | 含义 | 竞品对比 |
|--------|------|------|
| **专属** | 私有化部署，数据不出家门 | ChatGPT/Claude 数据上云端 |
| **越用越懂** | 三层记忆持续学习，不分人 | Siri 不记得上次聊什么 |
| **任何需求** | 技能包生态，社区贡献扩展 | 所有竞品功能固定不可扩展 |

### 差异化定位

| | ChatGPT/Claude | Siri/小爱 | **NexusAgent** |
|------|:--:|:--:|:--:|
| 归属 | 公司 | 公司 | **你的** |
| 数据 | 云端 | 云端 | **本地（不出家门）** |
| 记忆 | 遗忘 | 遗忘 | **三层记忆，越用越懂** |
| 能力 | 固定 | 固定 | **技能商店，无限扩展** |
| 付费 | 月租 | 免费 | **买断制** |

## 二、架构设计

### 部署架构

```
┌──────────┐      WiFi       ┌──────────────────┐
│ 手机/电脑  │◄──────────────►│ PC 或 Nexus Box   │
│ 只做终端  │  语音输入/输出  │                  │
│ 屏幕+麦   │               │  AI 大脑          │
│          │               │  记忆/推理/成长     │
└──────────┘               │  数据不出家门      │
                           └──────────────────┘
```

### 两阶段路线

| 阶段 | 产品形态 | 硬件要求 | 目标用户 |
|------|---------|---------|---------|
| 第一阶段 | PC 软件版 | 用户自有电脑 + 中端显卡 | 极客/开发者/重度 AI 用户 |
| 第二阶段 | Nexus Box | 专用硬件盒子（类似 Mac Mini） | 普通消费者 |

### 核心原则

- 一个软件，两种部署：PC 版和 Box 版跑同一套代码
- 手机不装 AI，手机只是遥控器
- 数据物理级隐私——永远不出用户家门

## 八、越用越懂——双重机制

### 记忆层（即时生效）
记住用户说过什么、喜欢什么。
- 三层记忆：短期(对话)→中期(MCP知识图谱)→长期(Git)
- 下次用户说"继续上次"→AI 知道上次聊到哪

### 微调层（逐步训练）——张凯提出
让 AI 的思维方式变成用户的形状。不是"记得你喜欢什么"，是"变成你的思考方式"。
- 技术：LoRA（低秩适配），只训练 ~200MB 的小层，不重训整个模型
- 数据来源：用户的对话、纠正、偏好、写作风格
- 效果：两三个月后，AI 的语气、思考角度、价值观磨成用户的样子
- 硬件：RTX 5080 16GB 用 QLoRA 完全跑得动

```
每个用户的 AI = 同一基座(Qwen2-VL 4.4GB)
                + 个人微调层(LoRA ~200MB)  
                + 个人记忆库(三层)
```

三者组合，张凯的 AI 和王律师的 AI 虽然同一个基座，但思考方式完全不同。

## 九、自主之心——AI自己学习自己

### 不是等人教，是自己知道不会，自己去学。

**三个核心机制：**

**一、自知之明**
AI能判断自己什么时候不行：
- 用户问了答不上来 → 标记，后台学
- 用户纠正了 → 最高优先级，立刻学
- 自己发现知识空白 → 空闲时补
- 网上出了新东西 → 定时扫描，增量更新

**二、自主学习（后台偷偷学）**
用户不用的时候（上班、睡觉），AI在后台：
1. 搜资料（课本、论文、教程）
2. 读理解（提取关键知识）
3. 做训练样本
4. LoRA微调
5. 验证（自己能讲清楚才算学会）

下次用户问同样的问题 → 比上次答得更好。

**三、Constitution约束**
学归学，不乱来。学什么、学完用在什么范围——Constitution画好边界。
不能因为学了物理就去炒股，不能因为看了法律书就假装律师。

### 和微调的区别
- 用户微调 → 变成"用户的形状"（风格、偏好、决策方式）
- 自主学习 → 填补"知识的空白"（不会的东西自己去学）
- 两者互补，各自独立运行
