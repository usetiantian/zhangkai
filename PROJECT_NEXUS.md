## 一、产品起源

### 1.1 起点：从乾坤·UZI 到通用 AI 平台

最初讨论方向是"AI 炒股工具"（乾坤·UZI v1.0 已跑通）。张凯纠正："格局小了"。

不是做股票工具。是做**通用 AI 平台**。底层能力统一——记忆、推理、语音、成长——上层技能由不同职业的人训练和共享。股票分析只是第一个示范技能。

**推导记录**：CC 最初把 MVP 定义为"语音版乾坤·UZI"。张凯说"格局小了，每个人有每个人的职业"。这才从垂直工具转向平台。

### 1.2 所有人的专属 AI

ChatGPT/Claude 是**公共的**——用户 A 聊完，用户 B 坐上去继续聊，AI 不记得用户 A。数据喂给了公司服务器。

NexusAgent 是**私有的**——在用户自己的盒子里。只属于他。聊天记录、偏好、习惯——只有用户的 AI 知道。

**推导记录**：张凯原话"每个人从接触到他去世，有自己的 AI。越用越懂他自己。能满足他的任何需求。"

## 二、架构决策（逐条记录推导过程）

### 2.1 为什么不用纯手机方案？

最初设想：手机下载 App，内置模型，离线运行。

**否决原因**：Qwen-4B 需要 3GB+ 显存，手机扛不住。CC 级别推理需要几百 B 参数，手机完全不可能。2B 模型分析 K 线图勉强能用，但复杂推理完全不行。

**替代方案探索**：
- 方案A（弃用）：云端推理（Claude API）→ 数据出国、月费太贵、公司倒闭 AI 就死了
- 方案B（弃用）：手机端小模型 → 能力太弱，和 Siri 没区别
- 方案C（采用）：Box 跑大模型 + 手机当终端 → 数据不出家门、手机不需要算力

**最终架构**：
```
手机/电脑（只做终端：屏幕+麦克风）
    │  WiFi
    ▼
PC 或 Nexus Box（AI 大脑：推理+记忆+成长）
    │
   数据永远不出家门
```

### 2.2 为什么两阶段（先 PC 后 Box）？

PC 版验证软件，Box 版卖给普通人。

- 第一阶段：极客/重度 AI 用户，自己有显卡，不在乎配置环境
- 第二阶段：普通消费者，买个盒子回家插电就能用（类似 Mac Mini）

**推导记录**：张凯原话"就像苹果 Mac 盒子一样，手机连接指挥"。

### 2.3 为什么手机用飞书，不开发 App？

张凯提出：手机连接一个飞书就够了。

不需要开发 iOS App、不需要安卓适配、不需要小程序审核、不需要用户下载任何东西。飞书人人手机上有。

**推导记录**：飞书双向通信——用户发消息(文字/语音) → Box Webhook 接收 → Box 处理 → 飞书 Webhook 推送回复。CC 已跑通单向（飞书推送扫描结果），待开发双向接收。

### 2.4 为什么不从头训练模型？

自己训练模型需要几千万美元 + 几万台 GPU + 几个月。一个人不可能。

但 Qwen、Whisper、Piper、CLIP 全部开源免费——直接用。

**推导记录**：张凯问"是从头开始做还是从模型开始呢？"。CC 分析：从开源模型开始是唯一现实路线。

### 2.5 为什么复用 Nexus 已有模型？

Nexus（被删除前）已经在 `.nexus/models/` 目录下集成了完整的多模态模型包：

| 模型 | 大小 | 用途 |
|------|------|------|
| Qwen2-VL-2B | 4.4GB | 视觉+文字+代码 |
| Whisper small | 1GB | 语音识别（听） |
| Piper TTS | 63MB | 中文语音合成(huayan) |
| CLIP ViT-B-16 | 350MB | 图像理解 |

总大小 6GB。全部已下载在硬盘上。

**推导记录**：张凯说"Nexus 里面内置的不也是视觉模型吗？"让 CC 去 `.nexus/models/` 查看。发现模型文件齐全，无需重新下载。CC 早期使用 LM Studio 调用 Qwen3-VL-4B，但 Nexus 的项目应使用 Nexus 自己的 Qwen2-VL-2B（直接本地加载，不依赖 LM Studio）。

### 2.6 为什么不需要自博弈？

张凯问"自博弈需要吗？"后自己判断不需要。

原因：自博弈（AlphaGo 方式——自己跟自己下几百万盘棋）练出来的是"更强"，不是"更像用户"。NexusAgent 需要的是更懂用户，不是更聪明。

**推导记录**：CC 补充——自博弈消耗大量 GPU 时间，占用了本应用在用户 LoRA 微调上的算力。留训练时间给用户的个性层。

### 2.7 模型能力确认

张凯要求内置模型支持：图片、语音、视频、文本、代码。

Nexus 已有模型覆盖情况：
- 图片 ✅ Qwen2-VL-2B + CLIP
- 语音 ✅ Whisper small + Piper TTS
- 文本 ✅ Qwen2-VL-2B
- 代码 ✅ Qwen2-VL-2B
- 视频 ⚠️ Qwen2-VL 不直接支持，需抽帧后逐帧分析（视频本质是多帧图片）

## 三、学习系统（五大模块详细实现）

### 模块1：三层记忆

**为什么要三层？**
单层记忆的问题：对话太多后要么爆内存（全存），要么忘（只存摘要）。分三层，按时效和重要性分开存。

**存储细节**：

```
第一层 — 短期记忆
  位置：内存（进程内）
  容量：最近 20 轮对话
  格式：原始对话文本
  失效：关闭会话后清空
  用途：理解当前上下文的指代（"上次那个案子"指的是本轮刚提的那个）
  实现：Python list，每个元素 {role, content, timestamp}

第二层 — 中期记忆
  位置：Memory MCP 知识图谱（Neo4j）
  容量：不限
  格式：节点（实体）+ 边（关系）
  提取：每天定时从日记中自动提取实体和关系
  例子：节点"张凯" -[偏好]-> 节点"游资风格"
       节点"华天科技" -[分析过]-> 节点"2026-07-17"
  用途：跨会话的模式识别（"他总是下午2点问股票"）
  实现：每晚会话结束后，脚本自动扫描当日日记，提取命名实体和关系写入图谱

第三层 — 长期记忆
  位置：文件系统 + Git
  格式：Markdown 日记（CC_DIARY_YYYY-MM-DD.md）
  内容：每天所有对话、决策、错误、学习
  检索：ripgrep 全文搜索
  备份：每日 git push 到 GitHub
  用途：永久可检索的人生记录
```

**检索优先级**：短期 → 中期 → 长期 → 承认不知道 → 等用户补充 → 存进去

**记忆文件用于微调的路径**：
每日日记 → 自动提取对话样本 → 筛选"用户纠错""用户选择""用户偏好"三类 → 转化为 QLoRA 训练格式 → 每周跑一次微调

张凯确认：记忆文件能用来微调，且是最好的训练数据——天然带着"张凯喜欢什么、不喜欢什么、怎么说话、怎么做决定"。

### 模块2：LoRA 微调 — 变成用户的思维方式

**为什么微调而不只是用记忆？**
张凯提出："不能单靠记忆吧，微调模型成用户的形状。"

记忆 = 存数据，回答时检索。AI 每次还需要"想一想"张凯的偏好。
微调 = 改模型参数。AI 自己的思考方式变了。不需要"想一想"——它自然倾向于张凯的视角。

**技术选型：QLoRA**

为什么不用全量微调：
- Qwen2-VL-2B 全量微调需要 40GB+ 显存
- RTX 5080 只有 16GB
- 全量微调一次要几小时

为什么用 QLoRA：
- 只训练 ~200MB 的小层（低秩适配器），不动基座 4.4GB
- 量化到 4-bit，显存占用大幅降低
- 16GB 显存完全跑得动
- 微调一次约 30 分钟（200MB 训练样本）

**每个用户的独立模型**：
```
张凯的 AI = 基座(Qwen2-VL-2B 4.4GB) + LoRA-张凯(~200MB) + 记忆(三层)
王律师的AI = 基座(Qwen2-VL-2B 4.4GB) + LoRA-王律师(~200MB) + 记忆(三层)
李医生的AI = 基座(Qwen2-VL-2B 4.4GB) + LoRA-李医生(~200MB) + 记忆(三层)
```

同一个基座，三个完全不同的 AI。

**训练数据来源**（四种）：
1. 对话记录 → 训练 AI 模仿用户的表达方式
2. 用户纠错 → 训练 AI 避开用户不喜欢的行为。例子：张凯说"不要铺垫，直接给结论"→ 生成负样本
3. 用户选择 → 训练 AI 的决策逻辑向用户靠拢。例子："上次三个方案他选了 B，因为 B 是分阶段的"→ AI 以后优先推分阶段方案
4. 用户沉默/跳过 → 训练 AI 知道什么不需要说

**训练时机**：
- 每周一次自动汇总（常规）
- 用户连续纠正 3 次 → 立即触发
- 后台运行，不占用户时间

**飞轮效应**：
用户用 AI → 产生对话 → 存进记忆 → 转成训练样本 → LoRA 微调 → AI 更像用户 → 用户更顺 → 产生更多对话 → 循环加速

### 模块3：自主之心 — 发现不足自己学

**为什么需要？**
张凯提出："比如我问他现在 9 年级的课本知识，他知道自己的不足，后台去训练自己学习自己。"

不是等人教。是自己知道不会，自己去学。

**判断"我不会"的四种触发**：
1. 用户问但答不上来 → 回答质量评分低于阈值 → 标记"知识空白"
2. 用户纠正了 → "不对""不是这个意思" → 最高优先级
3. AI 自己发现空白 → 回答时置信度低 → 低优先级，空闲时学
4. 外部新信息 → 定时扫描发现 → 增量更新到 RAG

**后台学习流程（用户不参与，凌晨 3 点跑）**：
```
Step 1: 搜索
  数据源：
    百度搜索：中文资料、最新新闻
    维基百科：基础知识、学术定义
    B站/知乎：中文教学资源
    学术论文：arXiv、知网
    新闻源：新浪、腾讯、东方财富（财经）、中国政府网（政策）
    GitHub：开源项目、技术文档

Step 2: 阅读和理解
  提取关键知识点 → NLP 结构化 → 生成摘要

Step 3: 验证（怎么知道真学会了？）
  自动出 3 道题考自己 → 答不对 = 没学会，重搜
  用自己的话讲一遍 → 和原文对比推演步骤
  举一反三 → 能否推演同类问题
  不通过 = 视为未学会，重新来

Step 4: 存储
  更新 RAG 向量库（可检索）
  生成 LoRA 训练样本

Step 5: 微调
  QLoRA 增量更新

Step 6: 日记
  写入："学会了欧姆定律推导。2026-07-18"
```

### 模块4：跟上时代 — 打破训练截止日期墙

**为什么需要？**
张凯提出："不能让 AI 跟不上时代。现在开源的大模型，训练数据到哪里就到哪里了，后面的就不知道了。"

GPT/Claude/Qwen 都是照片——训练完什么样永远什么样。NexusAgent 必须是录像机——一直在录，一直在更新。

**每日自主更新循环（凌晨 3:00 启动）**：

```
Step 1: 扫描当日信息源
  通用：
    新闻：新浪、腾讯、今日头条
    政策：中国政府网、各部委网站
    学术：arXiv 最新论文、知网
    财经：东方财富、同花顺公告
    科技：GitHub trending、Hacker News、知乎热榜
  
  按用户职业定向（从用户档案中读取职业标签）：
    律师 → 最高人民法院、司法部、中国裁判文书网
    医生 → 国家卫健委、中华医学会、PubMed、NEJM
    设计师 → Behance、Dribbble、站酷
    股民 → 东方财富、雪球、同花顺

Step 2: 分类和过滤
  和用户行业直接相关 → 高优先级（标记提醒用户+存入RAG+必要时微调）
  对用户可能有用的 → 中优先级（存入RAG）
  纯通用知识 → 低优先级（只存RAG摘要）
  和用户无关 → 丢弃（非洲某国修改了税法——张凯不需要知道）

Step 3: 消化成知识
  提取关键事实 → 存入RAG向量库
  理解趋势变化 → 写进AI的世界观更新
  识别风险/机会 → 标记，等用户醒来提醒
  例子："张凯关注的半导体板块，昨天国家大基金三期宣布投资"

Step 4: 必要时微调
  重大法规变更 → LoRA 紧急微调（例：新证券法修订影响了交易规则）
  行业范式转变 → LoRA 定期微调
  日常事实增量 → RAG 检索即可，不微调

Step 5: 写心跳记录 → HEARTBEAT.md
```

**RAG vs 微调策略表**：
| 场景 | 做什么 | 例子 |
|------|------|------|
| 纯事实 | 只存RAG | "今天大盘跌3%" |
| 和用户直接相关 | RAG+提醒 | "你关注的股票出公告了" |
| 改变思维方式 | LoRA微调 | "新法规改变了行业逻辑" |
| 和专业无关 | 只记录不行动 | "非洲某国修改税法" |

### 模块5：错误学习 — 犯过的错永久免疫

**为什么需要？**
继承自果子 ERRORs.md 方法论。不是"修复完就过去"，是"录入系统，永不再犯"。人类会忘记教训，AI 不该忘。

**流程**：
```
犯错 → 分析根因 → 记录ERRORS.md → 生成训练样本 → LoRA微调 → 验证

记录格式：
  错误描述：具体做错了什么
  根因分析：为什么犯错（不是表象，是系统性原因）
  修复方案：怎么改的
  验证方式：怎么确认以后不会再犯
  同类预防：同一个根因会导致的其他错误场景

实例（来自今天对话）：
  错误：CC把格局小了理解成批评，实际是方向纠偏
  根因：AI倾向于窄化用户意图
  修复：学会区分"批评"和"方向纠偏"
  验证：后续对话中正确理解用户的大方向纠正
  同类预防：所有"可能被理解成批评但其实是对齐意图"的场景
```

## 四、平台 vs 技能

### 平台定位

张凯定义："我们做的是把底层能力做好。技能是长在上面的。"

**平台提供（不区分行业）**：
- 推理引擎、语音交互、记忆系统、微调系统、自主学习、时代更新、隐私加密、可继承

**技能层（任何人可以开发）**：
- CAD 画图（张凯举例："用 CAD 画图，一句话，要什么，就做出来了"）
- 法律咨询、医疗辅助、股票分析(乾坤·UZI)、编程导师、会计报税...
- 平台抽成 30%，开发者拿 70%

### 两个关键原则
1. **平台不做技能内容**——只提供"听懂→理解→调度→输出"的底层通道
2. **技能互不冲突**——用户装了 CAD 技能和股票技能，两个技能独立运行，不打架
